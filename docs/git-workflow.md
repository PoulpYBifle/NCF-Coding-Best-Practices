# Git & GitHub — Workflow

## Règles de commit

- **Committer après chaque feature ou modification significative**
- Les commits sont liés aux hooks Husky — le pre-commit valide avant d'enregistrer

## Branches & Pull Requests

- **Grosse feature ou modification majeure** → créer une nouvelle branche → ouvrir une **PR**
- Les petits changements peuvent aller directement sur `main`

## Outils complémentaires

- **Claude Actions** / **GitHub Copilot** — à explorer pour l'automatisation CI/CD
